# Database Security Project

## Overview
This project demonstrates the design and implementation of a relational database using SQL Server. The goal was to transform raw academic data into a structured, normalized database while enforcing data integrity and relationships between entities.

## Technologies Used
- SQL Server Management Studio (SSMS)
- SQL
- Excel (CSV data sources)

## Key Components

### Database Design
- Created relational schema from raw datasets
- Defined primary keys for each table
- Established foreign key relationships between tables

### Normalization
- Converted raw data into normalized tables (1NF → 3NF)
- Reduced redundancy and improved consistency

### SQL Implementation
- Wrote SQL scripts for:
  - Table creation
  - Data insertion
  - Relationship enforcement
  - Querying and data retrieval

### Data Sources
- Imported structured CSV datasets into SQL Server
- Organized data across multiple related tables

## Security Relevance
This project highlights how proper database design contributes to security:
- Enforces data integrity through constraints
- Reduces redundancy and inconsistencies
- Supports controlled and structured data access
- Provides a foundation for secure database management

## Key Takeaways
This project strengthened my ability to design and implement relational databases while understanding the connection between data structure and security. It also improved my SQL skills and ability to manage real-world datasets.
